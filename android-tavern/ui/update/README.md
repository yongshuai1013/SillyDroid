# ui/update

## 模块职责

- 放应用内更新的 UI 协调层。
- 当前主要提供可复用于主界面 overlay 和设置页 about 区的统一更新协调逻辑。

## 主要公开入口 / 核心类

- `AppUpdateCoordinator`

## 上游依赖与下游被谁使用

- 上游依赖 `domain` 和 `core/ui`。
- 下游被 `feature/main` 与 `feature/settings` 复用。

## 修改时优先放这里的内容边界

- 更新按钮、角标、状态文案、下载完成后的安装器拉起协调。
- 同一条更新链在不同页面区域的 UI 复用。

## 不该放进来的内容

- 更新元数据抓取和 SHA 校验实现。
- 与更新无关的通用 UI 控制器。
- 主界面或设置页专属的大块业务逻辑。
